Baseline 설계 · 문제 정의 & 기본 구조

Shmoo metric 최적화 Baseline

  • 목표 — N>60개 binary 입력 X로 결정되는 rule-base metric Y를 최대화(또는 목표치 달성)하는 X 조합 탐색
  • 성격 — 비싸고 노이즈 있는 고차원 binary 배치 black-box 최적화
  • 접근 — 도메인지식 없는 일반 baseline 우선 → 이후 representation·constraint·prior·feature로 점진 주입
  • 예산 체급 — MAIN 320/일(15주 누적 ~2.4~3.4만, O(104)) · SUB 40/일(O(103))
MAIN 320/일 → O(10⁴) SUB 40/일 → O(10³) 탐색공간 2⁶⁰ ≈ 10¹⁸

문제 정의 — X 인자 · Y 인자 특징

입력 X binary N>60

  • 탐색공간 260 ≈ 1.15×1018
  • 그룹 — 여러 bit가 합쳐져 하나의 변량
  • 배타(mutex) — 일부는 서로 배타적(최대 1개)
  • 독립 — 일부는 서로 무관
  • 큰 상호작용 — 일부 변수쌍의 상호작용이 매우 큼

출력 Y 단일 metric

  • Shmoo array에서 rule-base로 파생되는 단일 metric
  • 예: width / height / area 및 이들의 조합
  • 측정 노이즈 존재 — 같은 X도 Y가 다를 수 있음
  • 목표 = Y 최대화 / 목표치(threshold) 달성
관점 — X 변화가 Y 변화에 영향을 주는 "최적화 가능한 black-box". 최적화 가능성 자체는 도메인 지식으로 정당화된다.

데이터 특징

  • 초기 데이터 0건 (cold start) — 첫 데이터부터 직접 생성. 첫 실험의 역할은 최적화보다 탐색공간 파악
  • 평가(생산) 비용 큼 · 배치 단위 — 설비 가동시간이 병목
    • MAIN: 40개/시간 × 8시간 = 하루 ~320개 · SUB: 하루 ~40개
  • 데이터 누적 가능 — 매 반복 모델을 갱신하는 순차 최적화 루프 성립
  • 기간 15주 고정 → 총 예산 상한
    • MAIN 누적 O(104) (~2.4~3.4만) · SUB 누적 O(103)
  • 측정 노이즈 — 반복측정·통계적 판정으로 다뤄야 함

제약조건 — 방법론을 결정하는 요인

  • 예산 ≪ 탐색공간 — O(104)도 260의 ~10-14 → 전수·무작위 불가 → 구조 활용 필수
  • 반복 wall-clock — 매일 배치 생성이 빨라야 → 데이터에 선형 확장되는 모델(트리/선형/진화) 선호, GP의 O(n³)·HMC 회피
  • 추론 지연 한도 — 다음 후보 생성 ≤ 3초(목표 ≤ 500ms), 학습은 무제한 → 무거운 탐색은 오프라인, 후보 방출은 즉시(후보 큐)
  • 구조 규칙 추후 주입 — 지금은 구조(상호작용·linkage)를 데이터로 발견하는 능력이 바람직
  • 성공 기준 = target threshold 통과 X 발견(satisficing) — 통과는 confirmation(반복측정 + LCB/guardband)으로 확정
    • 기록 지표: budget-to-target · threshold-hit count · 통과 X의 다양성

모형의 기본 구조 현 가설 · 데이터 확보 후 벤치로 확정

🟢 MAIN — O(10⁴)

모델기반 진화 + 대리모델 (SAEA)
  • 연관학습 EA(DSMGA-II / LT-GOMEA) — 변수 묶음(building block) 단위 탐색
  • 지도학습 대리모델(GBM/RF)로 후보 사전선별 + 다양성 확보

🔵 SUB — O(10³)

배치 베이지안 최적화 (BO)
  • BOCS형 희소회귀 / SMAC3 / TPE + screening DoE
  • 배치 추출 = Thompson sampling 등(중복 회피)

공통 골격 — 옵티마이저만 교체

인코딩 X0→X
rule-based transform→ DoE 시드
탐색공간 파악→ 순차 갱신 루프→ 후보 큐 서빙→ confirmatory 판정→ 도메인지식 점진 주입
예산 체급에 따라 — 구조학습 EA(중예산) vs 표본효율 BO(저예산). 공통적으로 변수 구조 활용이 핵심.
