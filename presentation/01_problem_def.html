Problem Definition · 01

Shmoo metric 최적화
Baseline — 문제 정의

N>60 binary 입력으로 결정되는 metric을, 제한된 실험 예산 안에서 최적화한다.
도메인지식 없는 일반 baseline을 먼저 세우고 — 이후 점진 주입한다.

MAIN · 320/일 → 누적 O(10⁴) SUB · 40/일 → 누적 O(10³) 대외비 제외 · 2026-06-22
한 줄 요약

N개(N>60)의 binary 입력 X로 결정되는 metric Y를 최대화(또는 목표치 달성)하는 X 조합을, 제한된 실험 예산 안에서 찾는다.

도메인지식을 하드코딩하지 않는 일반 baseline을 먼저 세우고, 구체 규칙은 이후 representation · constraint · prior · feature로 점진 주입한다.

예산 가정 — MAIN: 하루 320개(15주 누적 ~2.4~3.4만) · SUB: 하루 40개(누적 ~수천). 체급에 따라 전략이 달라진다.

문제 정의

입력 Xbinary 변수 N개 (N > 60)
출력 YShmoo array에서 rule-base로 파생되는 metric
예: width / height / area 및 이들의 조합 — 구체 정의는 대외비
목표Y를 최대화(또는 목표치 달성)하는 X 조합 탐색
탐색공간최소 260 ≈ 1.15 × 1018 (구조 제약으로 실제 가용 공간은 더 작음)

변수 간 구조

GROUP

MUTEX

⚠️ 구체 규칙은 대외비. baseline은 구조를 강하게 가정하지 않고, 추후 제약/사전지식으로 끼워넣을 수 있는 구조로 설계한다.
문제 관점 · Framing

왜 이게 풀 수 있는 문제인가

  • 데이터 0건에서 시작하지만, 도메인 관점에서 X 변화가 Y 변화에 영향을 주는 "최적화 가능한 black-box"로 본다.
  • 문제의 최적화 가능성 자체가 도메인 지식으로 정당화된다 무작위 잡음이 아니라 구조 있는 반응면이라는 전제
  • 초기 baseline은 구체 도메인지식을 하드코딩하지 않는다 구체 규칙은 representation·constraint·prior·feature로 점진 주입

예산 체급이 전략을 가른다

🟢 MAIN 320 / 일

O(10⁴)
15주 누적 ~2.4만 ~ 3.4만
  • 지도학습 대리모델 + 진화알고리즘이 본령에 진입하는 체급
  • 지배 요인 = 구조 활용 + 알고리즘 효율

🔵 SUB 40 / 일

O(10³)
설비 가용시간 축소 / 워밍업 구간
  • 표본효율이 지배하는 체급 (BO 계열)
  • 지배 요인 = 표본 효율
두 체급 모두 예산 ≪ 260 (MAIN도 ~10-14) → 전수·무작위 불가, 구조 활용이 본질.

성공 기준 · 1

탐색 objective ≠ 프로젝트 성공 기준

OPTIMIZER OBJECTIVE
Y 최대화 (maximize Y)
탐색은 단순·명확한 목적함수로 구동
PROJECT SUCCESS
Y_target 통과 X를 예산 내 발견
satisficing / feasibility 문제
따라서 best-Y만이 아니라 budget-to-target · threshold-hit count · 통과 X의 다양성을 함께 기록한다.
성공 기준 · 2 — 측정/판정 정책 (노이즈 대응)

루프는 단발, 통과는 confirmation

루프 중: deterministic single observation → Y_target 통과 후보만 → confirmation: 반복 측정 → 통계 기준으로 최종 판정
  • 왜? 단발 통과 선언은 winner's / optimizer's curse — 노이즈로 과대추정 → 재측정 시 평균 회귀
  • 판정규칙(baseline): one-sided LCB ≥ Y_target (= guardband, 95%) + 다중성 보정 "관측 평균이 넘었다"가 아니라 "참값이 신뢰수준으로 넘는다"
